# 8 - Why would you run the command 'git status'?
Command 'git status' lets users know if changes have been made to a file that have not yet been sent to the staging area yet. If changes have not been sent to the staging area the text will be red. If it has been sent to the staging area the text will be green. Once a commit has been completed the changes have been saved to local computer and 'git status' will indicate there is 'nothing to commit'.

# 9 - What does the command 'git add .' do?
Command 'git add .' sends unsaved changes to the staging area. Changes must be added to the staging area before they can "git commit -m 'message'" can be completed.  

# 10 - What is the command to push your committed changes to your repo on GitHub?
